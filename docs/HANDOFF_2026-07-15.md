## 4. 最新故障：只聊到遗体就发尸体照片

这是交接前最后处理、也最需要保留上下文的问题。

### 4.1 复现现场

最近一局《猩红文档》的玩家行动只是留在法伦办公室询问莱特。模型叙述和工具结果提到：

- 莱特死前变得多疑并更换门锁。
- 死亡证明由惠特克罗夫特签署，遗体仍在停尸房。

旧流程随即错误展示了莱特尸体照片，还把尸体证据写入线索清单；法伦提到其他人物时，
惠特克罗夫特、艾米莉亚和哈兰的头像也一并出现。

### 4.2 根因

故障不是模型单独误判，而是三条宽松链路叠加：

1. 素材触发器使用 `match_all` / `match_any` 扫描自由文本。
2. “莱特”与“遗体”命中后，图片显示逻辑反向写入了 `wright_body_evidence` 和
   `body_examined=true`。
3. NPC 素材分发扫描整个模组的人名，没有限制人物必须实际在当前场景。

这使“故事中提到某事实”错误地等价于“玩家已经亲眼发现该事实”。

### 4.3 已实现修复

当前实现采用“模型提出候选，确定性规则授权副作用”的混合方案，而不是继续增加关键词，
也不是把世界状态完全交给模型决定。

- `src/handouts.py`
  - 运行时不再使用 `match_all` / `match_any` 授权素材。
  - 素材只接受精确 `event + entity_id`。
  - 线索素材只按 `catalog_id` 或稳定线索 ID 解析。
- `tools/state_manager.py`
  - 取消自由文本自动附图。
  - `show-handout` 只有在对应目录线索已发现后才允许展示素材。
  - 阻止用素材名伪装成线索 ID 绕过门禁。
- `src/engine.py` / `src/agent_graph.py`
  - 删除“图片出现后反向提交线索/flag”的逻辑。
  - 模型和审计器提出的 `state_add_clue`、`sanity_event` 统一经过
    `_execute_model_tool`。
  - 带 `discovery_rules` 的目录线索，若未被本回合确定性发现预检授权，模型不能直接提交。
  - `sanity_trigger` / `sanity_event` 自由文本发图链路已移除。
- `src/turn_reconciler.py`
  - 审计器建议的线索同样经过门禁。
  - 提示明确：NPC 提到尸体、文件或地点不代表玩家已经直接发现对应证据。
- NPC/场景素材
  - 人物图片兜底只检查 `current_scene.npcs_present`。
  - 场景图片只检查当前场景，不再按全文扫描整个目录。
- 模组数据
  - `wright_body` 改为精确
    `clue_discovered + wright_body_evidence` 触发。
  - 密大办公室的 `npcs_present` 已修正为只有 `bryce_fallon`。

### 4.4 必须维持的单向契约

```text
玩家行动
  -> 确定性发现规则解析出稳定 clue_id
  -> 权威状态提交该线索
  -> 发出 clue_discovered(clue_id)
  -> 精确解析并显示对应素材
```

禁止反向链路：

```text
叙事提到关键词 -> 显示图片 -> 图片反向解锁线索/flag
```

SillyTavern 一类关键词 World Info 可以用于检索提示上下文，但不适合作为权威游戏状态的
提交机制。模型可以帮助理解自然语言或提出候选 ID，最终副作用仍必须通过模组声明的状态机、
发现规则和工具参数校验。

## 5. 当前权威回合流程

```text
WebSocket action
  -> server 连接/世界回合门禁
  -> EventStream.begin_turn(turn_id)
  -> GameEngine 确定性预检
  -> approach_text（若有）
  -> 骰点 / SAN / 状态迁移 / 精确线索发现
  -> Story 或 Combat Graph 流式叙事
  -> 模型工具候选经过统一授权门禁
  -> 结构化 choices
  -> TurnJournal 原子完成记录 + 自动存档
  -> FIFO 事件 done
```

断线时：

- 服务端请求取消仍在运行的旧模型流，避免旧世界锁长期占用。
- 已完成的 `TurnRecord` 可直接重放。
- active 回合先轮询；确认失败或中断后才回退到最近可靠快照。
- 前端通过 `turn_id/seq` 拒绝串线事件。
